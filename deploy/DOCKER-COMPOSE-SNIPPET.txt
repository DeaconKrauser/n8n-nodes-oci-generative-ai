Preferência profissional: NÃO editar os compose que já correm em produção.

Use os ficheiros de MERGE em deploy/ (copiar para ~/n8n e ajustar o nome do serviço):
  - docker-compose.oci-merge-main.EXAMPLE.yml
  - docker-compose.oci-merge-worker.EXAMPLE.yml

Fluxo (ajusta -f ao que o teu "docker compose config" aceitar):
  docker compose -f docker-compose-main.yml -f docker-compose.oci-merge-main.yml up -d
  docker compose -f docker-compose-worker.yml -f docker-compose.oci-merge-worker.yml up -d

Se o segundo comando falhar com: service "n8n" has neither an image nor a build context,
o worker.yml tem um serviço "n8n" incompleto ou precisa do main no mesmo comando:

  docker compose -f docker-compose-main.yml -f docker-compose-worker.yml -f docker-compose.oci-merge-worker.yml up -d

(ver deploy/ACTIVAR-NO-SERVIDOR.txt, passo 3.)

Montagem recomendada (imagem n8nio/n8n, user node): volume para
  /home/node/.n8n/custom/n8n-nodes-oci-generative-ai
e N8N_CUSTOM_EXTENSIONS com o mesmo caminho. Evita erro de execução
"Unrecognized node type: CUSTOM.*" quando /opt/... não é carregado em todos os contextos.
Com EXECUTIONS_MODE=queue, o worker (segunda linha "compose ... worker") precisa do mesmo
merge; só o main com extensão não basta para executar workflows.

Detalhes: deploy/README-PROFISSIONAL.txt

Deploy do código (PC de dev):
  ./scripts/deploy-via-ssh.sh ubuntu@IP:/home/ubuntu/n8n/custom-extensions/n8n-nodes-oci-generative-ai
  (rsync sem --delete por defeito; ver cabeçalho do script.)
